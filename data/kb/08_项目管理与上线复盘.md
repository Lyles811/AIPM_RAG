# 项目管理：从开发到上线你该做什么？

## 1. 里程碑（MVP 作品集建议）
- M1：PRD + MVP 拆分（1 天）
- M2：Baseline 跑通（2~3 天）
- M3：Demo 可体验（1 天）
- M4：评测脚本 + 报告（2 天）
- M5：上线链接 + 迭代记录（1 天）

---

## 2. 风险清单（AI 产品常见）
- 幻觉（胡编）
- 注入（知识库里带指令）
- 成本（token 失控）
- 时延（p95 太高）
- 版权/隐私（语料来源）

---

## 3. 上线 checklist（最小版）
- 有公开链接（或可本地复现）
- README 一键启动
- API Key 不泄露
- 有日志/反馈
- 有回归评测
- 有回滚策略（至少能关闭入口/切模型）

---

## 4. 复盘怎么写（写进简历更加分）
- 目标：想解决什么？
- 结果：指标如何？
- 问题：失败集中在哪里？
- 迭代：做了什么改动？为什么？
- 下一步：明确 v0.2 做什么
