# LLM / RAG / Agent 入门：三者是什么、怎么选？

## 1. LLM 应用：直接问答（不带外部知识）
适合：
- 通用写作、头脑风暴、总结
风险：
- 容易幻觉（编造细节）
- 对“最新/私有知识”不可靠

## 2. RAG：检索增强问答（推荐做作品集）
流程：
用户问题 → 检索知识库片段 → 把片段喂给 LLM → 生成答案 + 引用

优点：
- 可引用、可核验
- 私有知识可控更新
缺点：
- 检索质量决定上限
- 需要维护语料与索引

## 3. Agent：让模型“做事”
特点：
- 目标驱动（不是一句问答）
- 能规划步骤并调用工具（搜索、写文档、发请求）
风险更大：
- 权限、误操作、成本、注入更复杂

---

## 4. 怎么选？
- 你要“可信、可核验、基于固定资料回答” → RAG
- 你要“自动执行多步任务” → Agent（要更严格风控）
- 你只是“写一段文案” → 直接 LLM 就够
